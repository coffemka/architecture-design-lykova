# Вариант (по дипломной работе)

Преподаватель хочет унифицировать процесс планирования консультаций и ведения академической информации о студентах. Он стремится сократить количество ручной работы, избавиться от постоянных уточняющих сообщений от студентов и централизовать хранение учебных материалов.  
**Пользователи:** один преподаватель (основной пользователь системы), десятки студентов, которые взаимодействуют через Google Календарь.

## Требования

- преподаватель создаёт электронные карточки студентов  
- преподаватель прикрепляет к карточкам учебные материалы, ссылки, задания  
- преподаватель планирует временные слоты для консультаций  
- система автоматически обновляет расписание и синхронизируется с Google Календарём через публичную ссылку  
- студенты могут записываться на консультации через интерфейс Google Календаря  
- система автоматически генерирует инструкции студентам (как записаться)  
- преподаватель видит визуальный статус слотов («свободен» / «ожидание назначения»)  
- система автоматически учитывает академическую нагрузку, формируя статистику по проведённым консультациям и активности студентов  
- система отправляет напоминания о предстоящих консультациях и изменениях в расписании  

## Дополнительный контекст

Преподаватель вынужден заниматься автоматизацией, потому что число студентов растёт, и ручное управление консультациями приводит к частым накладкам во времени. Потеря информации о выполненных работах, ссылках и материалах создаёт хаос в учебном процессе.
